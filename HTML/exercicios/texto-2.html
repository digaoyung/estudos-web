MARCAÇÕES DE SEMÂNTICA

Temos as tags 'strong' e 'em' (de "emphasize", enfatizar -> ênfase).
Apesar de a tag, por padrão, formatar 'strong' como negrito, e 'em' como itálico, a diferença é que você pode definir um estilo para as tags, usando CS - ao contrário do 'b/i'.

OUTRAS TAGS

- tags de abraviação ('abbr'), outra tag para citação ('cite'), definição ('dfn'), endereço ('address'), correção ('del' + 'ins'), strike ('s')
Prof João da Silva em seu livro 200 Páginas Com Muito Conteúdo definiu que Café de qualidade é todo aquele tomado sem açúcar.

Visualmente, parece bobagem. Principalmente o strike, poque é, essencialmente, um tachado. Mas, são tags que interpretadores para pessoas com deficiência podem apresentar outras formas de leitura. Logo, como diz o título, são tags de semânticaa, mas no caso específico, é pra melhorar a semântica do interpretador.
No caso particular do strike, sinaliza que aquela informação está inválida (o 'del' mostra que "corrigiu")
E, visualmente falando, essas tags também podem receber um estilo CSS. Então, não seja preguiçoso / chorão, e escreve o teu código com uma semântica decente, de forma a facilitar - e incluir - os amiguinhos com deficiência.
